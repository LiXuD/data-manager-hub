# 权限控制系统设计方案

## 背景
当前数据管理平台权限控制薄弱，缺少业务用户-调用方关联、API接口级授权、权限管理等功能。

## 方案概述
采用方案 A：轻量 RBAC + 租户数据隔离 + 网关鉴权 + 前端动态菜单

### 1. 权限模型（RBAC）
| 模块 | 说明 |
|------|------|
| 权限管理 | 基于页面级 + 操作级，权限码格式如 `call:view`、`billing:edit` |
| 角色管理 | 角色绑定多个权限，如管理员、普通用户、只读用户 |
| 用户角色关联 | 用户可关联多个角色 |
| 用户调用方关联 | 用户可关联多个调用方，控制数据访问范围 |

### 2. 调用方 API 权限
| 模块 | 说明 |
|------|------|
| API Key 接口授权 | API Key 维度绑定可调用的接口，未授权接口返回 403 |
| 接口白名单 | 在 DataQueryController 中验证调用权限 |

### 3. 租户数据隔离
| 模块 | 说明 |
|------|------|
| 查询层强制过滤 | 各微服务 Service 层查询时强制带上 tenantId 参数 |
| 用户上下文 | UserContext 提供获取当前用户 tenantId 的方法 |

### 4. 网关统一鉴权
| 模块 | 说明 |
|------|------|
| Sa-Token 集成 | Gateway 集成 Sa-Token，统一拦截未登录请求 |
| 白名单配置 | `auth/login`、`health` 等免鉴权接口 |

### 5. 前端动态菜单
| 模块 | 说明 |
|------|------|
| 菜单权限过滤 | 登录后获取用户权限，动态隐藏无权限菜单项 |
| 按钮权限 | 页面内按钮根据权限隐藏/禁用 |

## 数据模型变更

### 新增表
- `permission`：权限表（页面/操作资源）
- `role_permission`：角色权限关联
- `user_caller`：用户调用方关联
- `api_key_interface`：API Key 接口授权

## API 变更
- `/permission/*`：权限 CRUD
- `/role/{id}/permissions`：角色权限管理
- `/user/{id}/callers`：用户调用方关联
- `/api-key/{id}/interfaces`：API Key 接口授权
